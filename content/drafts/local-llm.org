* Tests

I tested 4 local models and 2 cloud models with Ollama.

| Model          | Category | Description                                                                    |
|----------------+----------+--------------------------------------------------------------------------------|
| Kimi K2.6      | Cloud    | An open weight MoE model with 1 trillion parameters and 32b active parameters. |
| Gemini 3 Flash | Cloud    | Google's fast model.                                                           |
| Qwen3.6 27b    | Local    | A 27b dense model.                                                             |
| Qwen3.6 35b    | Local    | A 35b MoE model with 3b active parameters.                                     |
| Gemma4 26b     | Local    | A 26b MoE model with 4b active parameters.                                     |

For the tests, I did a short conversation about `podman`. All the answers seemed
generally correct. This first test is just a quick one to get a feel. In real
performance, I expect the prompt/input to be longer as smaller models need more
context. Larger models can be used as a "Google" of sorts, but smaller models
should aim to provide good reasoning given good context since less knowledge can
fit.

** Prompt

The test is a conversation with 3 prompts. The initial prompt starts with a
system like prompt to "Respond concisely" and asks for a list of podman
containers. The next prompt then asks for how to get the volumes. The final
prompt is more complex and asks to perform a backup and restore on a volume.

#+CAPTION: A system prompt and basic command.
#+BEGIN_SRC md
Respond concisely and with code snippets: List all podman containers
#+END_SRC

#+CAPTION: A follow up command.
#+BEGIN_SRC markdown
List all podman volumes
#+END_SRC

#+CAPTION: Something a little more complex and specific.
#+BEGIN_SRC markdown
```sh
podman volume ls
DRIVER      VOLUME NAME
local       forgejo-data
```

1. Back up the `forgejo-data` volume.
2. How can this volume be restored later?
#+END_SRC

** Response Quality

All responses were adequate. Since the Qwen models are small, they can't pack as
much information in. However, for `podman` all models did well. The final step
used a little bit of reasoning and models provided similar solutions. However,
Gemini 3 Flash may have known a little bit extra and it used the volume import
and export functionality.
